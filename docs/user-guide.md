## 1. 사전 요구사항

TalkBoard를 실행하기 위해 다음이 필요하다.

| 항목 | 요구사항 | 비고 |
|---|---|---|
| Node.js | 18 이상 | `node --version`으로 확인 |
| npm | Node.js에 포함 | `npm --version`으로 확인 |
| AI 계정 | ChatGPT, Gemini, Claude 중 1개 이상 | 무료/유료 모두 가능 |
| Claude CLI | (선택) | 브릿지 모드 사용 시에만 필요 |

---

## 2. 설치 및 실행

```bash
# 저장소 클론
git clone https://github.com/saintiron82/TalkBoard.git

# 디렉토리 이동
cd TalkBoard

# 의존성 설치
npm install

# 개발 모드 실행
npm run dev
```

실행하면 Electron 윈도우가 열리며, 상단에 컨트롤 바가, 하단에 AI 패널들이 나란히 표시된다.

## 3. 첫 실행 -- AI 서비스 로그인

앱을 처음 실행하면 각 AI 패널이 실제 웹사이트를 로딩한다. 토론을 시작하기 전에 각 서비스에 로그인해야 한다.

### 개별 로그인

각 패널에서 직접 로그인한다:

- **ChatGPT 패널**: chat.openai.com이 로딩됨 -> 로그인 클릭 -> 계정 정보 입력
- **Gemini 패널**: gemini.google.com이 로딩됨 -> Google 계정으로 로그인
- **Claude 패널**: claude.ai가 로딩됨 -> 이메일/Google 계정으로 로그인

### Google 계정 공유 로그인

여러 서비스에 같은 Google 계정을 사용하는 경우, 한 번의 로그인으로 모든 패널에 쿠키를 전파할 수 있다.

1. 상단 컨트롤 바의 **"G" 버튼** 클릭
2. Google 로그인 팝업이 표시됨
3. Google 계정으로 로그인 완료
4. 모든 패널에 쿠키가 자동으로 주입됨
5. 각 패널이 자동으로 리로드되어 로그인 상태가 반영됨

## 4. 슬롯 구성

슬롯은 토론에 참여하는 각 참가자(AI 또는 사용자)를 나타낸다.

### 기본 구성

앱 실행 시 기본으로 3개의 슬롯이 구성된다:

| 슬롯 | 타입 | 역할 |
|---|---|---|
| Slot 0 | ChatGPT | 기본 |
| Slot 1 | Gemini | 기본 |
| Slot 2 | Claude | 기본 |

### 슬롯 관리

| 조작 | 방법 |
|---|---|
| 슬롯 추가 | **"+" 버튼** 클릭. 같은 AI를 여러 개 추가 가능 (예: GPT 2개) |
| 순서 변경 | 슬롯을 **드래그**하여 원하는 위치로 이동 |
| 타입 변경 | 슬롯의 **타입 드롭다운**에서 AI 종류 선택 (ChatGPT / Gemini / Claude / User) |
| 지침 설정 | **"지침" 필드**에 역할이나 관점 입력 |
| 슬롯 제거 | **"x" 버튼** 클릭 (최소 1개는 유지해야 함) |

### 지침 예시

지침 필드를 활용하면 각 AI에게 특정 역할이나 관점을 부여할 수 있다:

- `"보수적 관점에서 반박하세요"`
- `"과학적 근거만을 기반으로 논증하세요"`
- `"일반 시민의 입장에서 쉽게 설명하세요"`
- `"상대방의 논점에서 논리적 오류를 찾아주세요"`

## 5. 토론 실행

### 토론 시작하기

1. **주제 입력**: 컨트롤 바의 입력란에 토론 주제를 작성한다.
   - 예: `"AI가 인간의 창의성을 대체할 수 있는가?"`

2. **모드 선택**: 실행 방식을 선택한다.

   | 모드 | 동작 | 추천 상황 |
   |---|---|---|
   | **Sequential** | 슬롯 순서대로 하나씩 실행 | 기본 추천. 각 AI가 이전 응답을 참고하여 논리적 흐름 형성 |
   | **Parallel** | 같은 AI 내 슬롯은 순서대로, 다른 AI는 동시 실행 | 빠른 실행이 필요할 때 |
   | **Reactive** | Sequential과 동일 | 향후 확장 예정 |

3. **라운드 수 선택**: 토론 반복 횟수를 선택한다.
   - `1R` -- 한 번만 (간단한 질의응답)
   - `3R` -- 세 번 (기본 토론)
   - `5R` -- 다섯 번 (심도 있는 토론)
   - `10R` -- 열 번 (장기 토론)

4. **"시작" 클릭**: 토론이 시작된다.

### 토론 진행 과정

토론이 시작되면 다음과 같이 진행된다:

```
[Round 1]
  Slot 0 (GPT):    주제에 대해 첫 응답 생성
  Slot 1 (Gemini): GPT의 응답을 참고하여 응답 생성
  Slot 2 (Claude): GPT + Gemini의 응답을 참고하여 응답 생성

[Round 2]
  Slot 0 (GPT):    Round 1의 Gemini, Claude 응답을 참고하여 반박/보완
  Slot 1 (Gemini): Round 1의 Claude 응답 + Round 2의 GPT 응답 참고
  Slot 2 (Claude): Round 2의 GPT, Gemini 응답 참고

[Round 3]
  ... (이하 반복)
```

각 슬롯이 실행될 때 해당 패널이 하이라이트 되어 어떤 AI가 응답 중인지 시각적으로 확인할 수 있다.

### 토론 제어

| 버튼 | 동작 |
|---|---|
| **중지** | 현재 토론을 일시정지. 진행 중인 캡처가 완료된 후 멈춤 |
| **이어서** | 일시정지된 토론을 중단된 지점부터 재개 |
| **새 주제** | 현재 토론을 완전히 리셋하고 새로운 토론 준비 |

## 6. 사용자 슬롯 (Human-in-the-loop)

AI들의 토론에 사용자가 직접 참여할 수 있다.

### 설정 방법

1. 슬롯의 타입 드롭다운에서 **"User"** 선택
2. 필요하다면 지침 필드에 자신의 역할 기술

### 사용 방법

토론이 진행되다가 사용자 슬롯의 차례가 오면:

1. 화면 하단에 **User Panel**이 활성화됨
2. AI들의 이전 응답을 참고하여 자신의 의견을 입력
3. **제출** 버튼 클릭
4. 입력 내용이 다른 AI들의 다음 컨텍스트에 포함됨

### 활용 예시

```
Slot 0: GPT     (찬성 측)
Slot 1: User    (중재자 역할 -- 논점 정리 및 질문)
Slot 2: Gemini  (반대 측)
Slot 3: Claude  (종합 평가)
```

사용자가 중간에 끼어들어 논점을 정리하거나, 특정 방향으로 토론을 유도하거나, 추가 질문을 던질 수 있다.

## 7. 검색

과거 토론 내용을 검색할 수 있다.

### 사용 방법

1. 슬롯 구성 영역 우측의 **검색 입력란**에 키워드 입력
2. **2글자 이상** 입력하면 자동으로 검색 시작 (300ms 디바운스)
3. 검색 결과가 드롭다운으로 표시됨

### 검색 범위

검색은 `~/DebateVault/topics/` 내의 모든 데이터를 대상으로 한다:

| 결과 유형 | 검색 대상 | 설명 |
|---|---|---|
| **주제 (topic)** | topic.json의 title, description | 토론 주제명 매칭 |
| **프롬프트 (prompt)** | round.json의 prompt | 각 라운드에서 사용된 프롬프트 매칭 |
| **캡처 (capture)** | gpt.md, gemini.md, claude.md, user.md | AI 응답 및 사용자 입력 본문 매칭 |

## 8. Claude CLI 브릿지

Claude 웹 패널 대신 `claude` CLI를 통해 Claude를 사용할 수 있다.

### 활성화

1. 컨트롤 바의 **"CLI" 체크박스** 활성화
2. Claude 슬롯이 웹 패널 대신 CLI를 통해 실행됨

### 전제 조건

- `claude` CLI가 시스템에 설치되어 있어야 한다
- 터미널에서 `claude --version`이 정상 동작하는지 확인

### 동작 방식

- CLI 활성화 시: `spawn("claude", ["-p", prompt], { cwd: sessionDir })` 형태로 실행
- 웹 캡처 실패 시 자동으로 CLI 폴백을 시도
- CLI 응답도 동일하게 `~/DebateVault/`에 저장됨

## 9. 데이터 관리

### 저장 위치

모든 데이터는 `~/DebateVault/`에 저장된다.

| 경로 | 내용 |
|---|---|
| `~/DebateVault/logs/` | 디버그 로그 파일 (날짜별) |
| `~/DebateVault/topics/` | 구조화된 토론 데이터 |

### 응답 저장 형식

각 AI의 응답은 개별 Markdown 파일로 저장된다:

- `gpt.md` -- ChatGPT 응답 원문
- `gemini.md` -- Gemini 응답 원문
- `claude.md` -- Claude 응답 원문
- `user.md` -- 사용자 입력 (사용자 슬롯이 있는 경우)

### 백업

`~/DebateVault/` 폴더를 통째로 복사하면 전체 데이터를 백업할 수 있다.

```bash
cp -r ~/DebateVault/ ~/DebateVault-backup-$(date +%Y%m%d)/
```

### 저장 경로 변경

환경 변수 `DEBATE_VAULT_PATH`를 설정하면 기본 저장 경로를 변경할 수 있다.

```bash
# 실행 시 환경 변수 지정
DEBATE_VAULT_PATH=/path/to/custom/vault npm run dev
```

## 10. 트러블슈팅

### "응답이 캡처되지 않아요"

- 해당 AI 서비스에 **로그인이 되어 있는지** 확인한다.
- AI 서비스의 웹 페이지가 업데이트되면 DOM 셀렉터가 변경될 수 있다. 이 경우 앱 업데이트가 필요하다.
- 패널을 수동으로 새로고침(F5)한 후 다시 시도한다.

### "구글 로그인이 안 돼요"

- 상단 컨트롤 바의 **"G" 버튼**을 클릭하여 팝업 로그인을 시도한다.
- 로그인 완료 후 각 패널이 **자동으로 리로드**되는지 확인한다.
- 팝업이 차단되는 경우, 시스템 설정에서 팝업 허용을 확인한다.

### "토론이 중간에 멈춰요"

- 30초간 DOM 변경이 없으면 **비활동 타임아웃**이 발생한다.
- 긴 응답의 경우 스트리밍이 감지되면 자동으로 **최대 5분까지** 대기 시간이 확장된다.
- 반복적으로 멈추는 경우, AI 서비스의 사용량 제한(rate limit)에 도달했을 수 있다.

### "검색 결과가 없어요"

- **2글자 이상** 입력해야 검색이 시작된다.
- `~/DebateVault/topics/` 디렉토리에 데이터가 존재하는지 확인한다.
- 첫 토론을 완료한 후에 검색이 가능하다.

### "앱이 실행되지 않아요"

- Node.js 버전이 **18 이상**인지 확인한다: `node --version`
- 의존성을 다시 설치한다: `rm -rf node_modules && npm install`
- 로그 파일을 확인한다: `~/DebateVault/logs/`
